Слайд 4 — Стайлинг Запорожца / Диагностика (90 сек)

Боль, с которой все знакомы. Говорить энергично, с примерами.

«Обвес сделать можно, но двигатель тот же — и он может выплюнуться посреди дороги.»

Боли текущей платформы:

  • 1С-Битрикс не обновлялся с 2019 — версия заморожена, техдолг растёт, безопасность под вопросом.
  • Мерчандайзинг на ручном приводе — Figma → текстовое поле в админке. Ни визуального редактора, ни глобальных правил. Каждый баннер — ручная работа.
  • Структуру каталога изменить невозможно — жёсткая схема Битрикса. Добавить атрибут = месяцы боли.
  • Монолит: одно изменение → каскадные баги — поправил корзину — сломал фильтры.
  • Костыли в интеграциях — Битрикс ↔ 1С ↔ Mindbox/Sailplay. Cron-скрипты, нет событийной модели.
  • Деградация поискового трафика — технические проблемы (LCP, robots.txt, битая карта сайта).
  • Данные клиентов разбросаны — часть в Битриксе, часть в 1С, часть в Mindbox. Единого профиля нет.
  • Низкая скорость блокирует трафик и конверсию — PageSpeed <50. Прямой удар по SEO и CR.
КЛЮЧЕВАЯ МЫСЛЬ

«Это не баги, которые можно починить. Это системные ограничения платформы. Запорожец не станет Теслой.»

Самый технический слайд. Показать схему, пройтись по уровням сверху вниз.

«Не латаем Запорожец. Проектируем с нуля.»

Пройтись по схеме:

  • Внешние системы — 1С ERP, Mindbox | Sailplay, Delivery, Payments, Yandex Cloud.
  • Anti-Corruption Layer — PIM защищает витрину от хаоса 1С. Запрещена прямая трансляция данных.
  • Модульный монолит OnlineStore — Laravel 13, PostgreSQL 16, Redis, Filament. Модули: Menu, Catalog, Lego, Promotions, SEO, Gifts.
  • API Gateway — единая точка входа для всех сателлитов.
  • Сателлиты — PIM, Content, Remains, CDP, Order, Payment, Geography, ClientRelated, Cart & Checkout, IDP. Отдельно деплоятся, изолированы по доменам.
  • Клиенты — Storefront (Nuxt 3 SSR), Mobile App, МПК (розница).
