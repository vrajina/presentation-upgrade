## Слайд 4 — Стайлинг Запорожца (Диагностика)

**Метафора: обвес можно поставить, но двигатель тот же — и он может выплюнуться посреди дороги.**

Ключевые боли текущей платформы:

1. **1С-Битрикс не обновлялся с 2020** — версия заморожена, техдолг растёт, безопасность под вопросом.
2. **Мерчандайзинг на ручном приводе** — Figma → текстовое поле в админке. Ни визуального редактора, ни глобальных правил отображения. Каждый баннер — ручная работа.
3. **Структуру каталога изменить невозможно** — жёсткая схема Битрикса. Добавить атрибут или изменить иерархию = months of pain.
4. **Монолит: одно изменение → каскадные баги** — всё связано со всем. Поправил корзину — сломал фильтры.
5. **Костыли в интеграциях** — Битрикс ↔ 1С ↔ Mindbox. Синхронизация на cron-скриптах, нет событийной модели.
6. **Деградация поискового трафика** — технические проблемы (LCP, robots.txt, битая карта сайта) привели к выпадению страниц из индекса Google.
7. **Данные клиентов разбросаны по разным системам** — часть в Битриксе, часть в 1С, часть в Mindbox. Единого профиля нет, CDP невозможен.
8. **Низкая скорость блокирует трафик и конверсию** — PageSpeed <50, высокий LCP. Это прямо бьёт по рекламе, SEO и CR.

**Тезис:** Всё это не фичи, которые можно починить. Это системные ограничения платформы. Запорожец не станет Теслой.

## Слайд 5 — Архитектура (Модульный монолит + Сателлиты)

**Не латаем Запорожец. Проектируем с нуля.**

Схема архитектуры (по Юре Шатилову):

- **Внешние системы:** 1С ERP, Mindbox, Delivery, Payments, Yandex Cloud — все внешние сервисы, с которыми мы интегрируемся.
- **Anti-Corruption Layer** — PIM как единственный барьер между «грязными» данными 1С и чистой витриной. Запрещена прямая трансляция данных из 1С.
- **Модульный монолит OnlineStore** (Laravel 13, PostgreSQL 16, Redis, Filament): ядро интернет-магазина — Menu, Catalog, Lego, Promotions, SEO, Gifts.
- **API Gateway** — единая точка входа для всех сателлитов.
- **Сервисы-сателлиты:** PIM, Content (CMS), Remains, CDP, Order, Payment, Geography, ClientRelated, Cart & Checkout, IDP — отдельно деплоятся, изолированы по доменам.
- **Клиенты:** Storefront (Nuxt 3 SSR), Mobile App (iOS/Android), Консультант (App для розницы).

**Ключевые принципы:**
- Не микросервисы — модульный монолит + сателлиты. Золотая середина.
- API-first: один BFF для веба, мобилки и приложения консультантов.
- Parallel Build → Cutover: строим параллельно, Битрикс в Feature Freeze, переключаем трафик одномоментно.

## Слайд 6 — Как звездолёт решает боли (Трансформация)

| Боль | Решение |
|------|---------|
| Мерчандайзинг вручную | PIM: визуальный редактор + глобальные правила |
| Каталог нельзя изменить | Good → GoodCard → SKU: гибкая атрибутная модель |
| Костыли интеграций | API-first: единый BFF для web + app |
| Урезанный ассортимент | Все склады, все остатки, real-time |
| Слепота по доставке | Прямая интеграция с логистикой |
| Деградация SEO | Nuxt 3 SSR + техническое SEO с нуля |
| Данные разбросаны | CDP: единый профиль клиента |
| Низкая скорость | SSR + CDN + оптимизация Core Web Vitals |

**Тезис:** Каждая боль — это не баг, а системное ограничение старой платформы. Новая архитектура закрывает их by design.

- SEO-стратегия + GEO-стратегия
- Прототипы UI и презентации — генерация + вёрстка

**Тезис:** AI не заменяет людей — он умножает. 3 человека с AI-стеком работают как команда из 10–15.

## Слайд 8 — Полётный план (Roadmap)

**Стратегия: Parallel Build → Cutover**

| Фаза | Содержание |
|------|------------|
| **Phase 1 — MVP** (активная) | PIM + CMS + Storefront BFF → новый фронт без Битрикса |
| **Phase 2 — Core Commerce** | Cart & Checkout, Account, Reviews, Notifications |
| **Phase 3 — Scale** | OMS, Loyalty, Pricing Engine, Returns |

- Битрикс в Feature Freeze. Новые фичи — только на новой платформе.
- Переключение трафика — одномоментно (Cutover). Никакого «скрещивания» систем.

**Тезис:** Рынок не ждёт — мы тоже. Phase 1 уже в работе.

## Ключевые мысли для запоминания

1. **Метафора Запорожца** — если двигатель старый, никакой обвес не поможет. Нужно строить заново.
2. **Модульный монолит** — не микросервисы (дорого и сложно для нашего масштаба), не монолит (негибко). Золотая середина.
3. **3 + AI** — маленькая команда может строить enterprise-платформу, если правильно использовать AI.
4. **Parallel Build → Cutover** — не мигрируем постепенно, а строим параллельно и переключаем разом.
5. **Anti-Corruption Layer** — PIM защищает витрину от хаоса 1С. Это архитектурный принцип, а не фича.
